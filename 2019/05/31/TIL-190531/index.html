Javascript - What I learn…

1. 프로미스란?

비동기 처리를 위해 하나의 패턴으로 콜백 함수를 사용한다. 그렇지만 콜백함수는 여러개의 로직을 한번에 처리하기 힘들고, 가독성이 떨어지고, 에러의 예외 처리가 곤란하다. 이런 단점을 해결하기 위해 ES6에서 만든 기능이 프로미스이다.

프로미스는 주로 서버에서 받아온 데이터를 화면에 표시한다.

2. 콜백 패턴의 단점

2.1 콜백 헬

자바스크립트는 비동기식 처리 모델은 병렬로 처리하여 다른 요청이 블로킹(blocking)되지 않는 장점이 있지만 여러개의 콜백 함수가 네스팅(Nesting)되어 복잡도가 높아지고 콜벡 헬(Callback Hell)이 발생한다. 즉 비동기 함수의 처리 결과를 가지고 다른 비동기 함수를 호출해야 할 때, 함수의 호출이 중복되고 복잡도가 높아지는 현상을 말한다.

복잡도가 증가하면 실수 및 에로 발생률이 증가하며, 이에 따른 에러 처리가 힘들다.

2.2 에러 처리의 한계

콜백 함수내에서 발생한 예외를 캐치하지 못하며, 이런 에러를 처리하지 못하는 문제점은 매우 심각하다. 여기서 예외(Exception)는 호출자 방향으로 전파된다.

3. 프로미스의 생성

프로미스는 생성자 함수를 통해 인스턴스화 한다. 이때 함수는 비동기 작업을 위해 콜벡 함수를 인자로 받는다.

상태 의미 구현
Pending 비동기 처리가 진행중 (처리 X) resolve, reject 함수가 아직 호출 되지 않은 상태
Fulfilled 비동기 처리가 성공 resolve 함수가 호출된 상태
Rejected 비동기 처리가 실패 reject 함수가 호출된 상태
Settled 비동기 처리가 수행된 상태 (성공 or 실패) resolve, reject 함수가 호출된 상태

4. 프로미스의 후속 처리 메소드

프로미스는 상태에 따라 다른 메소드를 체이닝 방식으로 호출한다. 이때 메소드(then, catch)를 통해 비동기 처리 결과, 에러 메세지를 반환한다.

then
두개의 메소드를 콜백 함수를 인자로 받는다. 두개의 함수는 성공과 실패시 반환하는데, 성공의 경우 fulfilled, resolve를 반환한다. 실패의 경우 rejected, reject가 반환된다.
then은 promise를 리턴하고, 프로미스는 then안에서 재 사용이 가능하다 => 프호미스 체이닝

catch
비동기 처리중 예외(실패)가 발생하면 호출된다.
